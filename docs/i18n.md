# 国际化（i18n）

WebNest / 网巢笔记 有两套相互独立的国际化层。

## 1. 应用 UI 层

实现于 `src/i18n/index.ts`。

- 单个 `messages` 对象，包含两种语言：`'zh-CN'`（简体中文，默认）和 `'en-US'`（英语），约 120 个文案键，覆盖全部 UI 字符串。
- 导出 API：

  | 名称 | 说明 |
  | --- | --- |
  | `t(key, variables?, locale?)` | 取文案，支持 `{var}` 插值 |
  | `getMessage` | 底层取值 |
  | `getCurrentLocale` / `setLocale` | 读取 / 设置当前语言 |
  | `getLocaleFromLanguage` | 由浏览器语言推断 locale |
  | `availableLocales` | 支持的语言列表 |
  | `localeLabels` | 显示名（`简体中文` / `English`） |
  | `useLocale()` | React hook，监听 `applocalechange` 事件强制重渲染 |

- **持久化**：当前语言存于 `localStorage` 的 `app-locale` 键。
- **自动检测**：未手动设置时，根据 `navigator.languages` 推断 —— 英语浏览器 → `en-US`，否则回退 `zh-CN`。
- **切换入口**：侧边栏底部的语言切换器。切换会触发 `applocalechange` 窗口事件，所有使用 `useLocale()` 的组件随之重渲染。

### 文案插值示例

```ts
t('workspaceStats', { sites: 3, pages: 12 });
// zh-CN → "3 个站点 · 12 个页面"
// en-US → "3 sites · 12 pages"
```

## 2. 清单 / 商店列表层

实现于 `public/_locales/`：

- `public/_locales/zh_CN/messages.json`
- `public/_locales/en/messages.json`

提供 `appName`（"WebNest"）与 `appDescription`，由 `manifest.json` 通过 `__MSG_appName__` / `__MSG_appDescription__` 引用，`default_locale: zh_CN`。这一层用于 Chrome 商店与扩展元信息的本地化，与应用内 UI 文案互不影响。

## 支持的语言

- 简体中文（默认）
- 英语

## 已知遗漏

部分硬编码中文字符串尚未纳入 i18n 系统，若需完整本地化可关注：

- `App.tsx` 的加载回退文案「正在加载工作区...」
- `LiveMarkdownEditor.tsx` 占位符「直接输入内容，所见即所得」「链接文字」
- `markdownInsert.ts` 的片段模板（如「## 标题」）
- `noteStats.ts` 的「暂无内容」「刚」「X 分钟前」等
- `Sidebar.tsx` 中的「更新标题失败」toast

## 新增文案

1. 在 `src/i18n/index.ts` 的 `messages` 对象中，为 `'zh-CN'` 和 `'en-US'` 同时添加同名键。
2. 组件中通过 `t('yourKey')` 使用，需要插值时传 `t('yourKey', { var: value })`。
3. 确保两种语言的键完全对齐，避免缺失键导致回退异常。
